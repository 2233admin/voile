# XAR-63: 产品版本分层设计

> 本文定义 Voile 的版本边界、开源/商业能力分层，以及免费到付费的升级路径。

---

## 1. 版本概览

| 维度 | 个人版（免费/开源） | 个人 Pro | 小团队版 | 专业版 |
|------|-----------------|---------|---------|------|
| 定价 | 免费 | ~15 USD/月 或 99 USD 买断 | ~30 USD/月/团队（≤5人） | 定价面议 |
| 部署 | 本地 | 本地 + 可选云备份 | 本地 + 共享服务器 | 私有部署 |
| 用户数 | 1 | 1 | ≤5 | 不限 |
| 数据主权 | 完全本地 | 完全本地 | 本地 + 可控服务器 | 自有服务器 |

## 3. 开源/商业边界原则

**开源部分（GitHub 公开）**：
- 全部 ingestion pipeline
- 存储 schema
- 基础写入 + 基础分析
- gRPC 协议定义（voile.proto）
- 插件 SDK（让社区能写 adapter）

**商业部分（闭源或 BSL 协议）**：
- 高级分析引擎（NER/关系图谱/决策回溯）
- 团队协作服务器组件
- 行业模板包 + Agent 角色包
- 云同步服务

**BSL（Business Source License）策略**：高级分析部分采用 BSL，4 年后转 Apache 2.0。既保护商业利益，又建立开源信任。

## 4. 免费 -> 付费的升级路径

### 触发点设计（不强迫升级，用价值驱动）

| 场景 | 免费版体验 | 升级后体验 |
|------|----------|----------|
| 话题历史 > 7 天 | 提示"历史已超出免费范围，Pro 无限制" | 完整历史 |
| 尝试查看人物关系图 | 显示节点但关系连线锁定 | 完整图谱 |
| 邀请第二个成员 | 提示需要团队版 | 多成员 vault |
| 导出报告 | 导出前 3 次免费 | 无限导出 |
| 设置第 3 个 Agent 角色 | 提示已到免费上限 | 无限角色 |

### 升级路径示意

```
开源自部署
  |-- 免费个人版（功能完整但有量级限制）
        |-- [触发点：需要历史深度/高级分析]
            --> 个人 Pro（买断 or 订阅）
                  |-- [触发点：需要协作]
                      --> 小团队版
                            |-- [触发点：规模化/合规]
                                --> 专业版 / 私有部署
```
