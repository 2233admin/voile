# 产品定位与价值主张

> Refs XAR-53

---

## 一句话定位

Voile 是本地优先的认知操作系统 -- 把聊天消息、链接、文档转化为结构化长期记忆，写入 Obsidian vault。

## 一段话定位

Voile（不动的大图书馆）是一套本地部署的知识归档系统。它从 QQ、微信、Telegram 等即时通讯工具实时接收消息，经过清洗、分析、结构化后，生成话题图谱、人物画像、决策日志和链接档案，持久化到 Obsidian vault 中。所有数据留在本地，所有推理过程可追溯，所有产出可被后续任务复用。

## 完整定位

### "本地优先认知操作系统" 的定义

| 关键词 | 含义 |
|--------|------|
| 本地优先 | 数据存储在用户本地（SQLite/Postgres + Obsidian vault），不依赖云服务，不上传原始聊天记录 |
| 认知 | 不只是存储 -- 包含话题提取、情感分析、人物画像、决策追踪等分析能力 |
| 操作系统 | 提供统一的消息输入、存储、分析、查询接口，各层可独立替换和扩展 |

### 核心价值

| 价值 | 说明 |
|------|------|
| 长期记忆 | 聊天消息不再是一次性信息流 -- 经过结构化后成为可检索的知识资产 |
| 结构化沉淀 | 原始消息被编译为话题图谱、画像卡片、决策时间线等有意义的知识单元 |
| 跨任务复用 | 沉淀的知识通过 Obsidian 双链和 gRPC 查询接口被其他工具和工作流消费 |
| 证据可追溯 | 每条知识单元都链接回原始消息，支持"这个结论从哪来的"式溯源 |

### Voile 是什么

- 聊天消息的结构化归档管线
- 本地部署的知识编译器
- Obsidian vault 的自动化写入器
- 跨平台（QQ/微信/Telegram）的消息统一入口
- 面向个人和小团队的长期记忆基础设施

### Voile 不是什么

| 误解 | 事实 |
|------|------|
| 聊天 AI / 对话机器人 | Voile 不回复消息，不参与对话。它是后台归档系统 |
| 大而全的 Agent 平台 | 不做通用任务执行、不做自治决策、不做工作流编排 |
| 云服务 / SaaS | 不提供托管服务，不上传数据到任何第三方 |
| Obsidian 插件 | Voile 是独立系统，Obsidian 是它的输出目标之一，不是宿主 |
| 搜索引擎 | 提供基础查询能力，但核心价值在"编译"而非"检索" |

## 架构与定位的对应关系

| 架构层 | 职责 | 对应价值 |
|--------|------|----------|
| Go gateway | QQ/微信实时接入、链接抓取、统一 API | 跨平台统一入口 |
| Python core | Schema 定义、DB 存储、分析 Agents、Obsidian 写入 | 结构化沉淀 + 证据追溯 |
| Rust kernel | ANN 向量检索、批量文本清洗 | 高性能处理保障 |

## 竞品区分

| 维度 | Voile | 通用笔记工具 | 聊天备份工具 | Agent 平台 |
|------|-------|-------------|-------------|-----------|
| 输入源 | QQ/微信/Telegram 实时流 | 手动输入 | 聊天记录导出 | API 调用 |
| 处理能力 | 话题/情感/画像/决策分析 | 无 | 无 | 通用任务执行 |
| 输出格式 | Obsidian vault 结构化文档 | 富文本笔记 | 原始聊天记录 | 任务结果 |
| 数据位置 | 本地 | 本地或云 | 本地 | 云端为主 |
| 核心用途 | 长期记忆编译 | 即时记录 | 历史备份 | 自动化执行 |
