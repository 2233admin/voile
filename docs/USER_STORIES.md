> Refs XAR-61

## 用户故事列表

### US-01: 群聊消息自动入库

| 维度 | 描述 |
|------|------|
| 角色 | 独立开发者 |
| 目标 | 不手动操作，群聊消息自动进入系统 |
| 触发动作 | 在 QQ 技术群发送/接收消息 |
| 期望结果 | 消息在 3 秒内出现在 Voile 数据库中，保留发送者、时间、群组、内容 |
| 场景分类 | **高频 + 高价值** -- 系统运转的基础，每天持续发生 |

### US-02: 历史消息批量导入

| 维度 | 描述 |
|------|------|
| 角色 | 研究员 |
| 目标 | 把过去一年的微信群聊记录导入系统 |
| 触发动作 | 用 WeChatMsg 导出 CSV，运行 `python -m core.importers.wechat_import` |
| 期望结果 | 10 万条历史消息在 10 分钟内导入完成，与实时消息共享同一 schema，可混合查询 |
| 场景分类 | **低频 + 高价值** -- 系统冷启动必需，只做一次但决定用户是否留下 |

### US-03: 链接自动归档

| 维度 | 描述 |
|------|------|
| 角色 | 内容工作者 |
| 目标 | 群里有人分享的文章链接自动保存正文 |
| 触发动作 | 群聊消息中出现 URL |
| 期望结果 | 系统自动抓取网页正文，存入数据库，生成 Obsidian 归档笔记，笔记中包含来源消息引用 |
| 场景分类 | **高频 + 高价值** -- 信息密集群组每天数十条链接，手动保存不现实 |

### US-04: 按话题回顾群讨论

| 维度 | 描述 |
|------|------|
| 角色 | 小团队负责人 |
| 目标 | 看到今天群里讨论了哪些话题，每个话题的结论是什么 |
| 触发动作 | 一天结束后查看 Obsidian vault 中当天生成的话题笔记 |
| 期望结果 | 每个话题一篇笔记，含标题、参与者、摘要、关键发言引用、标签分类 |
| 场景分类 | **高频 + 高价值** -- 核心产出，每天产生 |

### US-05: 搜索特定讨论

| 维度 | 描述 |
|------|------|
| 角色 | 独立开发者 |
| 目标 | 找到上个月关于"Redis 选型"的群讨论 |
| 触发动作 | 在 Obsidian 中搜索关键词"Redis 选型"，或通过 gRPC 查询接口 |
| 期望结果 | 返回相关话题笔记列表，点进去能看到完整讨论上下文和当时的结论 |
| 场景分类 | **高频 + 高价值** -- 知识复用的核心入口 |

### US-06: 追溯决策过程

| 维度 | 描述 |
|------|------|
| 角色 | 小团队负责人 |
| 目标 | 项目复盘时找到"为什么当时选了技术方案 A"的完整讨论 |
| 触发动作 | 在 Obsidian vault 中按项目标签浏览决策日志 |
| 期望结果 | 决策条目包含：决策内容、讨论参与者、支持/反对论点、最终结论、原始消息链接 |
| 场景分类 | **低频 + 高价值** -- 复盘/审计时触发，价值极高 |

### US-07: 了解某人的专业背景

| 维度 | 描述 |
|------|------|
| 角色 | 咨询顾问 |
| 目标 | 见客户前快速回忆"张三在群里聊过什么、关注什么领域" |
| 触发动作 | 在 Obsidian 中打开张三的人物卡片 |
| 期望结果 | 人物卡片含：活跃群组、高频话题、专业领域标签、关键发言摘要、最近互动时间 |
| 场景分类 | **中频 + 高价值** -- 社交和业务场景强需求 |

### US-08: 跨群话题汇总

| 维度 | 描述 |
|------|------|
| 角色 | 研究员 |
| 目标 | 看到"AI 监管"这个话题在不同群的讨论视角 |
| 触发动作 | 在 Obsidian 中搜索标签 `#AI监管`，查看关联笔记 |
| 期望结果 | 多个群组中关于同一话题的讨论被串联，能看到不同群体的观点差异 |
| 场景分类 | **中频 + 高价值** -- 研究场景核心需求 |

### US-09: 导出结构化数据

| 维度 | 描述 |
|------|------|
| 角色 | 独立开发者 |
| 目标 | 用脚本批量处理 Voile 的话题数据（如统计词频、生成报告） |
| 触发动作 | 通过 gRPC 接口或直接查询 SQLite/Postgres |
| 期望结果 | 数据以标准格式可访问，schema 稳定，文档完整 |
| 场景分类 | **低频 + 中价值** -- 开发者的长尾需求 |

### US-10: 一键部署运行

| 维度 | 描述 |
|------|------|
| 角色 | 所有用户 |
| 目标 | 不折腾环境，快速跑起来看到效果 |
| 触发动作 | `docker compose up` 或按 DEV_SETUP.md 操作 |
| 期望结果 | 30 分钟内完成部署，发一条 QQ 消息能在 Obsidian 里看到对应笔记 |
| 场景分类 | **低频 + 极高价值** -- 决定用户是否留下的第一印象 |
